# Tests

Snapshot tests via `run_tests.sh`. Each query runs against the fixture data dir, the output is normalised (strip `meta.started_at`, `meta.duration_ms`, `meta.tool_version`, `meta.args.out`; sort keys; indent 2), and the result is diffed against the committed expected file.

## Running

```bash
export OBLIVION_CONFLICTS_XEDIT="/c/path/to/TES4Edit_patched.exe"
./tests/run_tests.sh
```

Requires `bash` and `python` (3.x). Both are available on Git Bash for Windows when Python is installed via scoop / choco / python.org.

## Updating snapshots

After intentional output changes:

```bash
UPDATE_SNAPSHOTS=1 ./tests/run_tests.sh
git diff tests/fixtures/expected/   # review carefully
```

The runner never auto-updates without `UPDATE_SNAPSHOTS=1`. Always read the diff before committing.

## Fixtures

`fixtures/data/` contains a hand-built 3-plugin test corpus (`Master.esm`, `OverrideA.esp`, `OverrideB.esp`) generated by `fixtures/_generator/generate_fixtures.pas`. See that directory's README for the regeneration procedure - it's a manual GUI step, not run by `run_tests.sh`.

`fixtures/_generator/patch_smoketest.pas` proves the patched xEdit binary still works headlessly. Run separately if the binary is rebuilt:

```bash
"$OBLIVION_CONFLICTS_XEDIT" -IKnowWhatImDoing -autoload -autoexit \
  -D:tests/fixtures/data \
  -P:tests/fixtures/loadorder.txt \
  -script:tests/fixtures/_generator/patch_smoketest.pas
```

## Expected fixture results

For reference, the snapshot tests assert these high-level invariants:

| Query | Args | Result count | Notable |
|---|---|---|---|
| `query_list` | `--target=OverrideA.esp` | 3 | TestSword (caConflict, winner=B), TestArmor (caOverride, winner=A), TestDagger (caOverride, deleted, winner=A); TestChest correctly excluded |
| `query_record` | `--formid=00001001 --formid=00001002` | 2 | TestSword 3-link chain; TestArmor 2-link chain; per-subrecord status approximate (struct subrecords report caNoConflict due to empty edit_value) |
| `query_between` | `--a=OverrideA.esp --b=OverrideB.esp` | 1 | TestSword only (the only record both plugins directly override) |
